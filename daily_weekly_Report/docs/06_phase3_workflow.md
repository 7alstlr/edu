# 페이즈 3: Make.com 워크플로우 구성 가이드

Google Sheets의 데이터를 자동으로 Google Docs 보고서에 채우고 Outlook으로 발송하는 완전한 워크플로우를 구성합니다.

## 📋 작업 개요

**목표**: 매주 금요일 10:00에 자동으로 주간 보고서 생성 및 이메일 발송
**예상 시간**: 60분
**필수 요구사항**: 
- Make.com 계정 (완료)
- Google 계정 연동 (완료)
- Google Sheets 모듈 추가 (완료)

---

## 단계별 진행

### Step 1️⃣: 트리거 설정 (매주 금요일 10:00)

현재 상태: Make.com 시나리오 열려 있음

#### 1-1: 현재 Google Sheets 모듈 앞에 트리거 추가

1. Google Sheets 모듈의 **왼쪽에 있는 흰 동그라미** (입력 포인트) 위에 마우스 오버
2. **"+"** 버튼 또는 **"Add trigger"** 클릭

#### 1-2: Schedule 트리거 선택

1. 앱 검색 창에서 **"Schedule"** 또는 **"Scheduler"** 검색
2. **"Schedule"** 선택
3. **"At a specific time"** 또는 **"At a specific date and time"** 선택

#### 1-3: 트리거 설정

다음과 같이 설정:

```
Frequency: Weekly (매주)
Day of week: Friday (금요일)
Time: 10:00 (10시)
Timezone: Asia/Seoul (한국 표준시)
```

#### 1-4: Save

설정 완료 후 **"Save"** 클릭

---

### Step 2️⃣: Google Sheets 모듈 수정 (범위 설정)

현재 추가된 Google Sheets 모듈을 수정합니다.

#### 2-1: Google Sheets 모듈 클릭

캔버스에 있는 Google Sheets 모듈 클릭

#### 2-2: 설정 수정

다음 항목이 표시되어야 합니다:

```
Connection: My Google connection ✅
Spreadsheet: 일일업무일지_응답 ✅
Sheet: 응답 ✅
Range: [설정 필요]
```

**Range 설정**:
- 범위 입력: `A2:E100` 또는 `A:E`
- 또는 "Header row" 옵션 활용

#### 2-3: Save

---

### Step 3️⃣: 이미지 처리 (선택사항 - 나중에)

이미지를 처리하려면:
1. Google Drive 모듈 추가
2. "Download a file" 작업 선택
3. File ID 매핑

(현재는 스킵하고 나중에 추가 가능)

---

### Step 4️⃣: Google Docs 템플릿 복제

Google Sheets 모듈 다음에 **Google Docs 복제 모듈** 추가

#### 4-1: 새 모듈 추가

1. Google Sheets 모듈의 **오른쪽 아래** (출력 포인트)의 **"+"** 클릭
2. **"Google Docs"** 또는 **"Google Drive"** 검색

#### 4-2: Google Docs 모듈 선택

**"Copy a document"** 또는 **"Duplicate a document"** 선택

#### 4-3: 설정

```
Connection: My Google connection
Folder: 새 문서를 저장할 폴더 선택 (또는 My Drive)
Title: {{작성기간}} - {{담당자명}}의 주간 보고서
```

#### 4-4: Save

---

### Step 5️⃣: 데이터 입력 (Google Docs 업데이트)

복제된 문서에 데이터를 자동으로 입력합니다.

#### 5-1: Google Docs 모듈 추가

1. 이전 모듈의 오른쪽 **"+"** 클릭
2. **"Google Docs"** 검색
3. **"Replace text in a document"** 또는 **"Update a document"** 선택

#### 5-2: 설정

문서 ID와 치환할 텍스트를 설정합니다:

```
Document ID: [이전 모듈의 Document ID 선택]

Replace operations (치환 목록):
- {{작성기간}} → [주차 데이터]
- {{담당자명}} → Google Sheets의 담당자 필드
- {{작성날짜}} → TODAY() 또는 현재 날짜
- {{업무_진행_사항}} → Google Sheets의 업무 필드
- {{이슈_장애_우려}} → Google Sheets의 이슈 필드
- {{다음주_중요업무}} → Google Sheets의 다음주 필드
- {{개선사항}} → Google Sheets의 개선사항 필드
```

#### 5-3: Save

---

### Step 6️⃣: Outlook 이메일 발송

최종 단계: 생성된 보고서를 이메일로 발송합니다.

#### 6-1: Outlook 모듈 추가

1. 이전 모듈의 오른쪽 **"+"** 클릭
2. **"Outlook"** 또는 **"Office 365 Outlook"** 검색
3. **"Send an email"** 선택

#### 6-2: 설정

```
Connection: Outlook 계정 선택

To: {{사용자이메일}}
CC: (비워두기)
BCC: (비워두기)

Subject: [주간 보고서] {{작성기간}} - {{담당자명}}

Body (본문):
안녕하세요,

이번 주 주간 보고서가 생성되었습니다.

첨부된 Google Docs 링크를 통해 확인하실 수 있습니다.
{{구글독스링크}}

감사합니다.

Attachment: (선택사항 - PDF 파일)
```

#### 6-3: Save

---

## ⚙️ 모듈별 데이터 매핑

### Google Sheets → 변수 매핑

Google Sheets 모듈에서 읽은 데이터를 다음과 같이 사용:

```
Column A (작성 날짜): 1.A
Column B (담당자명): 1.B
Column C (업무 진행 사항): 1.C
Column D (이슈/장애): 1.D
Column E (다음주 업무): 1.E
```

각 모듈에서 이 변수들을 사용하여 동적으로 데이터 입력 가능

## 🧪 테스트 및 활성화

### Step 7️⃣: 워크플로우 테스트

#### 7-1: 수동 실행 테스트

1. Make.com 시나리오 페이지에서 **"Run once"** 또는 **"Execute once"** 클릭
2. 각 모듈이 성공적으로 실행되는지 확인
3. Google Drive에 새 문서가 생성되는지 확인
4. Outlook에 이메일이 발송되는지 확인

#### 7-2: 오류 확인

실행 후:
- ✅ Google Docs 복제 성공?
- ✅ 데이터 입력 완료?
- ✅ 이메일 발송됨?

오류가 있으면 각 모듈 클릭하여 로그 확인

#### 7-3: 결과 확인

Google Drive와 Outlook에서:
- Google Docs: 새 문서가 생성되고 데이터가 입력되었나?
- Outlook: 이메일이 수신되었나?

---

## 🚀 자동화 활성화

워크플로우가 정상 작동하면:

#### 8-1: 시나리오 활성화

1. Make.com 시나리오 페이지 상단의 **"Turn on"** 또는 **토글 스위치** 클릭
2. 상태가 **"ON"** 또는 **"활성화됨"**으로 변경

#### 8-2: 스케줄 확인

자동으로 **매주 금요일 10:00**에 실행됨

#### 8-3: 모니터링

이후 Make.com 대시보드에서:
- 실행 기록 확인
- 성공/실패 로그 추적
- 필요시 문제 해결

## 🚨 트러블슈팅

### 문제 1: Google Docs 복제 실패

**원인**: 권한 부족 또는 폴더 선택 안 함

**해결**:
1. 템플릿 Google Docs의 공유 설정 확인
2. Make.com의 Google Drive 권한 재확인
3. 대상 폴더 명확히 선택

---

### 문제 2: 데이터 입력 안 됨

**원인**: 플레이스홀더 형식 오류 또는 데이터 매핑 잘못됨

**해결**:
1. Google Docs의 플레이스홀더가 {{정확히}} 이 형식인지 확인
2. Google Sheets 모듈의 데이터 매핑 확인
3. "Replace text" 모듈에서 "Find" 부분이 정확히 일치하는지 확인

---

### 문제 3: 이메일 안 발송됨

**원인**: Outlook 연결 실패 또는 이메일 주소 오류

**해결**:
1. Outlook 계정 재인증
2. "To" 필드의 이메일 주소 확인
3. Make.com의 Outlook 권한 확인
