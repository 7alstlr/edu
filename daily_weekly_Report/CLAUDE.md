# CLAUDE.md

이 파일은 Claude Code(claude.ai/code)가 이 저장소에서 작업할 때 참고할 가이드를 제공합니다.

## 프로젝트 개요

**일일 업무 일지 자동화 도구** — 매일 작성하는 일일 업무 일지를 자동으로 수집하여 주간 보고서로 생성하고 이메일로 전송하는 자동화 시스템입니다.

상세한 요구사항은 [prd.md](prd.md)를 참조하세요.

## 기술 스택

### 데이터 수집 및 저장
- **입력 양식**: Google Forms (구글 계정 필수)
- **데이터 저장소**: Google Sheets (자동 응답 저장)
- **대체 옵션**: Notion, Microsoft Forms + Excel Online

### 자동화 플랫폼
- **주 플랫폼**: Make.com (Zapier 대체 가능)
- **기능**: 주간 데이터 수집, 보고서 생성, 이메일 발송 스케줄링
- **트리거**: 매주 금요일 10:00 실행

### 보고서 생성 및 이메일
- **보고서 형식**: Google Docs (템플릿 기반 자동 생성)
- **파일 포맷**: PDF 또는 Google Doc 링크
- **이메일 클라이언트**: Outlook (Microsoft 365 또는 개인 계정)

## 개발 로드맵 (단계별)

전체 개발 기간: 약 2-3시간 (단계별로 나누어 진행 권장)

### **페이즈 1: 입력 양식 및 데이터 저장소 설정** (~20분)

#### 1-1: Google Forms 생성 (8분)
- **목표**: 일일 업무 일지 입력을 위한 구글 폼 생성
- **가이드**: [📖 01_google_forms_setup.md](docs/01_google_forms_setup.md)
- **빠른 시작**: [⚡ 00_phase1_quickstart.md](docs/00_phase1_quickstart.md) 참고
- **작업 내용**:
  - Google Forms에서 새 양식 생성
  - 필드 추가:
    - 날짜 (날짜 선택, 자동 입력)
    - 담당자 (단문 입력)
    - 완료한 업무 (장문 입력)
    - 첨부 이미지 (파일 업로드)
  - 폼 제목 및 설명 작성
  - 폼 링크 생성 및 공유 설정
  - 링크를 [docs/form_url.txt](docs/form_url.txt)에 저장
- **완료 기준**:
  - ✅ 구글 폼이 생성되고 모든 필드가 추가됨
  - ✅ 테스트 응답 1-2개 제출 가능 확인
  - ✅ form_url.txt에 폼 링크 저장됨

#### 1-2: Google Sheets 응답 저장 설정 (7분)
- **목표**: Google Forms의 응답이 자동으로 Google Sheets에 저장되도록 설정
- **가이드**: [📖 02_google_sheets_setup.md](docs/02_google_sheets_setup.md)
- **빠른 시작**: [⚡ 00_phase1_quickstart.md](docs/00_phase1_quickstart.md) 참고
- **작업 내용**:
  - Google Forms에서 "응답" 탭 클릭
  - Google Sheets 아이콘 클릭하여 응답 연결
  - 새 스프레드시트 생성 또는 기존 시트에 연결
  - 스프레드시트 이름: `일일업무일지_응답`
  - 각 컬럼이 올바르게 매핑되었는지 확인 (Timestamp, 날짜, 담당자, 업무, 이미지)
  - 링크를 [docs/sheet_url.txt](docs/sheet_url.txt)에 저장
- **완료 기준**:
  - ✅ Google Sheets가 자동 생성되거나 연결됨
  - ✅ 테스트 응답이 시트에 저장되는지 확인
  - ✅ sheet_url.txt에 시트 링크 저장됨

#### 1-3: 입력 양식 테스트 (5분)
- **목표**: 실제 데이터 입력이 제대로 작동하는지 검증
- **가이드**: [📖 03_testing_setup.md](docs/03_testing_setup.md)
- **빠른 시작**: [⚡ 00_phase1_quickstart.md](docs/00_phase1_quickstart.md) 참고
- **작업 내용**:
  - 구글 폼에 테스트 응답 2-4개 제출
  - 이미지 파일 첨부 테스트 (최소 1개는 이미지 포함)
  - Google Sheets에 모든 데이터가 올바르게 저장되는지 확인
  - 파일명, 타임스탠프, 텍스트 모두 검증
  - 결과를 [docs/test_results.md](docs/test_results.md)에 정리
- **완료 기준**:
  - ✅ 모든 테스트 응답이 정확하게 저장됨
  - ✅ 이미지 업로드 및 저장 동작 확인
  - ✅ 한글 텍스트가 정상 저장됨
  - ✅ test_results.md에 테스트 결과 정리됨

### **페이즈 2: 보고서 템플릿 작성** (~40분)

#### 2-1: 템플릿 구조 설계 (10분)
- **목표**: 주간 보고서의 레이아웃 및 구성 요소 설계
- **작업 내용**:
  - 보고서 포맷 결정 (Google Docs)
  - 섹션 구성:
    1. 헤더 (작성자, 작성 주차, 작성 날짜)
    2. 주간 요약 (핵심 업무 3-5줄)
    3. 일일 업무 상세 내역 (월~금, 각 요일별 이미지+설명)
    4. 피드백/메모 섹션
  - 플레이스홀더 정의 (예: {{작성자}}, {{주차}}, {{요약}})
- **완료 기준**:
  - ✅ 문서 구조와 레이아웃 결정됨
  - ✅ `docs/template_structure.md`에 템플릿 구조 정리

#### 2-2: Google Docs 템플릿 작성 (25분)
- **목표**: 실제 Google Docs 문서에 템플릿 작성
- **작업 내용**:
  - Google Docs에서 새 문서 생성
  - 문서명: `주간 보고서 템플릿 - {{주차}}`
  - 템플릿 작성:
    - 헤더: 회사/부서 로고, 제목 스타일링
    - 메타데이터: 작성자 {{AUTHOR}}, 주차 {{WEEK}}, 날짜 {{DATE}}
    - 주간 요약 섹션 생성
    - 월~금 각 요일별 섹션 생성:
      - 요일 제목
      - 이미지 영역 (너비 제한)
      - 업무 내용 텍스트 영역
    - 스타일링: 폰트, 색상, 여백 통일
  - 스타일 가이드 적용 (읽기 쉬운 폰트, 일관된 제목 스타일)
- **완료 기준**:
  - ✅ 전체 템플릿이 작성되고 형식이 정렬됨
  - ✅ 모든 변수/플레이스홀더가 {{}} 형식으로 명확하게 표시됨
  - ✅ 문서 ID를 `docs/template_id.txt`에 저장

#### 2-3: Make.com 연동 설정 (5분)
- **목표**: 문서를 Make.com에서 사용할 수 있도록 공유 및 설정
- **작업 내용**:
  - Google Docs 공유 설정 (링크 공유, 누구나 접근 가능)
  - 또는 특정 서비스 계정에 편집 권한 부여
  - 문서 ID 확인 및 저장 (URL에서 추출)
  - Make.com에서 사용할 인증 정보 확인
- **완료 기준**:
  - ✅ 문서가 Make.com에서 접근 가능하도록 권한 설정됨
  - ✅ 템플릿 문서 ID가 저장됨

### **페이즈 3: Make.com 워크플로우 구성** (~60분)

#### 3-1: Make.com 초기 설정 및 Google 연동 (10분)
- **목표**: Make.com 계정 생성 및 Google 서비스 연동
- **작업 내용**:
  - Make.com 회원가입 (https://www.make.com)
  - 새 시나리오(Scenario) 생성
  - Google 계정 연동:
    - Google Sheets 연결 (API 인증)
    - Google Docs 연결 (API 인증)
    - Google Drive 연결 (이미지 다운로드용)
  - 각 연동의 정상 작동 확인
- **완료 기준**:
  - ✅ Make.com 계정 생성 완료
  - ✅ Google 서비스 3개 모두 인증됨
  - ✅ Make.com 시나리오 ID를 저장

#### 3-2: 트리거 및 데이터 수집 모듈 (12분)
- **목표**: 매주 금요일 10:00에 워크플로우 시작, Google Sheets에서 주간 데이터 읽기
- **작업 내용**:
  - 모듈 1: Schedule 트리거
    - 매주 금요일 10:00에 작동하도록 설정
  - 모듈 2: Google Sheets 모듈
    - 스프레드시트 선택: `daily_work_logs`
    - 시트 선택: 응답 시트
    - 조건: 해당 주(금요일 기준)의 모든 데이터 읽기
    - 필터: 날짜 범위 설정 (월요일~금요일)
  - 데이터 매핑 확인 (각 컬럼이 정확히 읽혀지는지)
- **완료 기준**:
  - ✅ 트리거가 설정됨
  - ✅ 데이터 읽기 모듈이 정확한 데이터를 반환하는지 테스트됨
  - ✅ 테스트 실행으로 예상 데이터가 반환되는지 확인

#### 3-3: 이미지 다운로드 모듈 (10분)
- **목표**: Google Sheets에 저장된 이미지 파일 다운로드
- **작업 내용**:
  - 모듈 3: Google Drive 이미지 다운로드
    - 각 응답의 이미지 URL/파일 ID 추출
    - Google Drive에서 파일 다운로드
    - 로컬 스토리지 또는 임시 저장소에 저장
  - Iterator 모듈 추가 (여러 응답 처리)
  - 에러 처리: 이미지가 없는 경우 처리
- **완료 기준**:
  - ✅ 테스트 이미지 다운로드 성공
  - ✅ 여러 이미지 동시 처리 가능 확인

#### 3-4: Google Docs 템플릿 복제 모듈 (12분)
- **목표**: 매주 새로운 Google Docs 문서 생성 (템플릿 복제)
- **작업 내용**:
  - 모듈 4: Google Docs 복제 모듈
    - 템플릿 문서 ID 지정
    - 새 문서명: `주간 보고서 - 2024년 1월 1주차 - {{작성자}}`
    - Google Drive의 특정 폴더에 저장 설정
    - 복제된 문서의 ID를 다음 단계로 전달
- **완료 기준**:
  - ✅ 템플릿 복제 성공
  - ✅ 새 문서가 Google Drive에 생성됨
  - ✅ 문서 ID가 정확하게 전달됨

#### 3-5: 데이터 입력 모듈 (12분)
- **목표**: 복제된 문서에 수집한 데이터 자동 입력
- **작업 내용**:
  - 모듈 5: Google Docs 콘텐츠 업데이트
    - 이전 단계에서 받은 문서 ID 사용
    - 텍스트 치환 (플레이스홀더를 실제 데이터로 변경):
      - {{AUTHOR}} → 담당자명
      - {{WEEK}} → 현재 주차
      - {{DATE}} → 작성 날짜
      - {{SUMMARY}} → 주간 요약
      - {{MONDAY}}, {{TUESDAY}} 등 → 각 요일 업무 내용
    - 이미지 삽입:
      - 각 요일 섹션에 대응하는 이미지 삽입
      - 이미지 크기 조정 (일관된 너비/높이)
- **완료 기준**:
  - ✅ 데이터 입력이 정확하게 됨
  - ✅ 이미지가 올바른 위치에 삽입됨
  - ✅ 최종 문서가 보기 좋게 형식화됨

#### 3-6: PDF 변환 및 이메일 발송 준비 (4분)
- **목표**: 생성된 Google Docs를 PDF로 변환 (선택사항)
- **작업 내용**:
  - 모듈 6: PDF 변환 (선택사항)
    - Google Docs를 PDF로 내보내기
    - PDF를 Google Drive에 저장
  - 또는: Google Docs 링크 그대로 사용
  - 최종 문서 URL/링크 저장
- **완료 기준**:
  - ✅ PDF 또는 링크가 준비됨
  - ✅ 다음 단계(이메일 발송)에서 사용할 수 있는 상태

#### 3-7: Make.com 워크플로우 통합 테스트 (10분)
- **목표**: 3-1~3-6 모듈을 모두 연결하여 전체 워크플로우 테스트
- **작업 내용**:
  - 모든 모듈을 순서대로 연결
  - 에러 처리 경로 추가 (실패 시 알림)
  - Make.com 시뮬레이터로 전체 흐름 테스트 (실제 실행 아님)
  - 각 모듈의 입출력 데이터 확인
- **완료 기준**:
  - ✅ 모든 모듈이 정확하게 연결됨
  - ✅ 시뮬레이션 실행 후 예상 결과 확인
  - ✅ 에러가 없거나 에러 처리 로직 추가됨

  - 테스트 데이터 준비:
